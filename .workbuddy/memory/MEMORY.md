# 塔罗牌分销系统 - 项目记忆

## 项目概述
塔罗牌多级分销系统（网页版），包含两个页面：分销系统（管理员+代理统一登录）和客户商城。

## 技术栈
- 后端：Node.js + Express + better-sqlite3 + JWT
- 前端：原生 HTML/CSS/JS（SPA 模式，hash 路由）
- 数据库：SQLite（文件：data/tarot.db）
- 端口：3000

## 核心业务规则
- **三级分销**：管理员 → 一级代理 → 二级代理
- **佣金抽成限制**：一级代理从二级代理佣金中抽成 ≤ 30%（即二级佣金 ≥ 一级佣金 / 1.3）
- **价格隔离**：代理只能看到佣金金额，看不到商品真实售价
- **邀请码体系**：每个代理一个邀请码，客户用码进商城，下级代理用码注册
- **下单通知链路**：客户下单 → 直接代理 + 上级代理 + 管理员 均收到通知
- **发货流程**：管理员录入快递单号 → 代理收到通知 → 代理转发给客户

## 默认账号
- 管理员：admin / admin123
- 一级代理：agent1 / 123456（邀请码 TAROT8A1，佣金20%）
- 二级代理：agent2 / 123456（邀请码 TAROT8B2，佣金15%）

## 页面结构（2个页面）
- public/index.html - 入口引导页（2个入口：分销系统 + 客户商城）
- public/system.html - 分销系统SPA（管理员+代理统一登录，根据角色显示不同功能；含代理注册页）
- public/shop.html - 客户商城SPA（邀请码进入→浏览→下单）
- public/css/style.css - 公共样式
- server.js - 后端主服务（含统一登录接口 /api/auth/login）
- db.js - 数据库初始化与种子数据
- test-flow.js - 完整业务流程测试脚本

## 启动方式
```bash
node server.js
```
访问 http://localhost:3000
