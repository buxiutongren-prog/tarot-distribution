# 2026-07-04 工作日志

## 塔罗牌分销系统开发（完整版）

### 完成内容
从零搭建了完整的塔罗牌多级分销系统（网页版），涵盖客户商城、代理后台、管理员后台三大模块。

### 技术选型
- Node.js + Express + better-sqlite3 + JWT
- 前端原生 HTML/CSS/JS，SPA 模式
- SQLite 文件数据库，零配置启动

### 核心功能实现
1. **客户商城**：邀请码验证 → 商品浏览 → 商品详情 → 下单（收货信息填写）→ 成功页
2. **代理后台**：登录 → 仪表盘（佣金/订单统计）→ 订单管理（只看佣金不看价格）→ 佣金明细 → 邀请码/链接管理 → 下级代理管理（一级代理设佣金，30%抽成限制）→ 消息通知
3. **管理员后台**：登录 → 仪表盘（全局统计+7天趋势图）→ 商品CRUD → 代理管理（创建一级代理/设佣金/禁用）→ 订单管理（发货录入快递单号）→ 消息通知
4. **二级代理注册**：通过一级代理邀请码自助注册

### 关键业务逻辑
- 佣金计算：直接佣金 = 售价 × 代理佣金率；间接佣金 = 售价 × (一级佣金率 - 二级佣金率)
- 抽成限制校验：(一级佣金率 - 二级佣金率) / 二级佣金率 ≤ 30%
- 价格隔离：代理API不返回 product_price / total_price 字段
- 通知链路：下单/发货时通知直接代理 + 上级代理 + 管理员

### 测试结果
完整业务流程测试通过（test-flow.js），包括：
- 客户下单 → 代理收通知 → 管理员发货 → 代理收快递单号
- 佣金正确分配（128元商品：二级代理19.20元，一级代理6.40元）
- 30%抽成限制正确拦截（10%被拒，16%通过）
- 价格隔离验证通过

## 邀请链重构（核心逻辑变更）

### 用户需求
- 管理员只需要看到自己邀请码，不需要看到所有代理的邀请码
- 代理也一样，只看自己的
- 佣金/可见性基于"谁邀请谁"，不是基于角色等级
- 小明（一级代理）邀请的小李的出单小明可见；小红不是小明邀请的，小红出单和小明无关

### 具体改动
1. **数据库种子**：管理员新增 commission_rate=0.25，拥有唯一邀请码 ADMIN0001
2. **登录**：管理员登录时返回 invite_code 和 commission_rate
3. **注册 API**：支持管理员邀请码→一级代理、一级代理邀请码→二级代理
4. **管理员创建代理**：parent_id 改为管理员真实ID（不再硬编码为1）
5. **佣金链逻辑**：从固定的 level1→level2 改为通用的逐级遍历（for循环沿 parent 链上溯）
6. **下单通知链**：同样改为逐级上溯，覆盖完整邀请链
7. **发货通知链**：同样改为逐级上溯
8. **邀请码管理页面**：管理员只看到自己的邀请码+直邀代理列表，不再看所有代理
9. **重新生成邀请码**：支持 admin 自己重新生成
10. **代理列表 API**：增加 parent_id 字段

- ✅ 管理员只看到直邀代理（张一级），看不到间接代理（李二级）
